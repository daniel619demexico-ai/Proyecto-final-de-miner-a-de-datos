Minería de Datos · KDD

Minería de Datos

Extracción no trivial de patrones implícitos, previamente desconocidos y potencialmente útiles desde grandes conjuntos de datos.

Proceso KDD (Knowledge Discovery in Databases)

  • Selección: Integración de fuentes relevantes.
  • Preprocesamiento/Limpieza: Ruido, outliers, valores nulos.
  • Transformación: Reducción, discretización, atributos derivados.
  • Minería de Datos: Algoritmos para hallar patrones.
  • Interpretación/Evaluación: Visualización y validación del conocimiento.

Tareas y Técnicas · Correlación

Clasificación (árboles, SVM) · Clustering (k-means, DBSCAN) · Asociación (Apriori, FP-Growth) · Regresión · Anomalías.
Correlación directa entre tarea y técnica: clustering no supervisado → k-means; clasificación supervisada → Random Forest.

Almacén de datos & Vistas Minables Retail Analytics

Se implementó un Data Warehouse en arquitectura estrella (Hecho Ventas, dims: Tiempo, Producto, Cliente, Sucursal).
ETL construido con Talend + Python integrando datos CRM y transaccionales. A partir del DW se generaron 3 vistas minables listas para minería:

  • vista_segmentacion_clientes (edad, antigüedad, ticket promedio, frecuencia)
  • vista_cesta_compra (productos por transacción) → reglas asociación
  • vista_tendencia_ventas (serie temporal + indicadores macro)
Vistas materializadas en PostgreSQL + Cubos MOLAP

Regresión Lineal Árboles CART K-means ++ Apriori Random Forest
  • Segmentación no supervisada: Clustering K-means (4 clusters: "Premium", "Leales", "Ocasionales", "Potenciales").
  • Reglas de asociación (Apriori): Descubrimiento de productos complementarios (soporte 0.03, confianza 0.75).
  • Predicción churn: Árbol de decisión + Random Forest (accuracy 89%, AUC 0.94).
  • Pronóstico ventas trimestral: Regresión multivariante (R² 0.87) usando vistas minables con atributos de estacionalidad.
